# Web Serial Debugger

Web Serial Debugger 是一个基于浏览器 Web Serial API 的网页串口调试助手，可用于 USB 串口设备的调试。

推荐使用 Chrome 或 Edge 浏览器。当前版本只支持 USB 串口调试，不支持蓝牙。

## 运行方式

不要把直接双击 `index.html` 作为主要测试方式。建议通过本地 HTTP 服务器运行：

1. 进入项目目录。
2. 执行：

```bash
python -m http.server 8000
```

3. 在 Chrome 或 Edge 中打开：

```text
http://127.0.0.1:8000/index.html
```

## V1 功能

- 串口连接 / 断开
- 波特率选择，默认 `115200`
- 串口接收显示
- 文本发送
- 发送结尾选择：`None` / `LF` / `CRLF`
- HEX 显示
- HEX 发送
- `[TX]` / `[TX HEX]` 发送记录
- 自动滚动
- 显示时间戳
- 暂停接收显示
- 清空接收区
- 保存日志为 TXT
- 状态栏提示

## 使用步骤

1. 启动本地服务器：`python -m http.server 8000`
2. 打开网页：`http://127.0.0.1:8000/index.html`
3. 选择波特率。
4. 点击“连接串口”。
5. 在浏览器弹窗中选择 COM 设备。
6. 在输入框中输入内容并发送。
7. 需要保存记录时，点击“保存日志”。

## 常见问题

### 打开后显示目录列表怎么办

确认浏览器地址是否打开到了项目目录根路径。请访问：

```text
http://127.0.0.1:8000/index.html
```

### 浏览器不支持 Web Serial API 怎么办

请使用 Chrome 或 Edge 浏览器。部分浏览器不支持 Web Serial API，页面会提示当前浏览器不支持。

### 点击连接后提示 No port selected 怎么办

这通常表示在浏览器串口选择弹窗中取消了选择，或没有选中任何串口设备。请重新点击“连接串口”，并选择正确的 COM 设备。

### 为什么 HEX 发送时不追加换行

HEX 发送用于发送二进制字节。为了避免破坏二进制数据，开启 HEX 发送后不会追加 `None` / `LF` / `CRLF` 中的任何发送结尾。

### 为什么打开 file:/// 路径不建议使用

Web Serial API 更适合在安全上下文中测试。使用本地 HTTP 服务打开页面更接近实际网页运行环境，也更方便排查浏览器权限和资源加载问题。
